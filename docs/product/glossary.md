# Product Glossary

## 用語定義

| 用語 | 意味 |
|---|---|
| 今月 | Asia/Tokyo基準の現在日が属する月 |
| 今日 | Asia/Tokyo基準の現在日 |
| 使える収入 | `included_in_balance = true` の収入 |
| 貯める収入 | `included_in_balance = false` の収入 |
| 固定費 | `start_month` 以降の月に反映され、`is_active = true` のもの |
| 出費 | トップ画面から金額のみで登録する支出 |
| 今月残り使える金額 | 使える収入から固定費と出費を引いた金額 |
| 年間実収支 | 年間全収入から年間固定費と年間変動費を引いた金額 |
| 年間生活費残り | 年間使える収入から年間固定費と年間変動費を引いた金額 |

金額はすべて日本円の整数として扱う。

---

## 残額計算

今月残り使える金額は以下で計算する。

```txt
今月残り使える金額
= 今月の使える収入合計
- 今月の固定費合計
- 今月の出費合計
```

今月はAsia/Tokyo基準で判定する。

月次検索では、対象月の月初以上、翌月月初未満を対象にする。

---

## 収入の扱い

収入は複数登録できる。

ボーナスや貯金したい臨時収入などは、「収入として記録するが、今月使える金額には含めない」ことができる。

そのため、収入には `included_in_balance` フラグを持たせる。

```txt
true:
  今月使える金額に含める

false:
  収入として記録するが、今月使える金額には含めない
```

---

## 固定費の扱い

固定費は毎月同じ金額で発生するものを基本とする。

固定費には開始月を持たせ、開始月以降の月次計算に反映する。

MVPでは固定費の終了月は持たない。

終了した固定費は無効化して対応する。

MVPでは、固定費を無効化すると過去月の集計からも除外される。

過去月の固定費履歴を厳密に残す対応は将来対応とする。

---

## 出費の扱い

出費は金額のみで登録できる。

登録日時はサーバー側の現在日時を使用する。

MVPでは出費カテゴリを持たない。

出費登録後は3秒間Undoできる。

Undoは最後に登録した出費1件のみ対象とする。

Undoは出費削除として扱う。

Undo対象がなくなった後は、同じUndo操作で追加削除しない。

---

## UUIDとDBの扱い

各テーブルのIDはDB側でUUIDを生成する。

PostgreSQLの `gen_random_uuid()` を利用する方針とし、Go側では通常の新規作成時にUUIDを明示的に生成しない。

詳細は `../db/spec.md` を参照する。

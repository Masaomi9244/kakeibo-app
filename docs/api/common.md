# API Common Spec

## 共通仕様

すべてのAPIは `/api` 配下に置く。

すべての主要APIは認証必須とする。

```txt
Authorization: Bearer <access_token>
```

---

## 日付パラメータ

月指定は `YYYY-MM` 形式とする。

```txt
2026-05
```

日指定は `YYYY-MM-DD` 形式とする。

```txt
2026-05-01
```

---

## タイムゾーン

集計・表示の基準は Asia/Tokyo とする。

月次検索では以下の範囲に変換する。

```txt
対象月の月初 00:00:00 以上
翌月の月初 00:00:00 未満
```

---

## レスポンス命名

JSONレスポンスはキャメルケースで返す。

DBカラム名のスネークケースをそのまま返さない。

---

## 成功レスポンス形式

成功レスポンスは必ずJSON objectで返す。

配列、文字列、数値、booleanをトップレベルで直接返さない。

### 作成・更新

作成APIと更新APIは、対象リソースをリソース名のキーで包んで返す。

```json
{
  "income": {
    "id": "uuid",
    "amount": 250000,
    "incomeDate": "2026-05-25",
    "memo": "給料",
    "includedInBalance": true
  }
}
```

出費登録のように画面更新に集計値が必要な場合は、同じレスポンスに必要な最小集計を同梱する。

```json
{
  "expense": {
    "id": "uuid",
    "amount": 780,
    "spentAt": "2026-05-01T10:30:00+09:00"
  },
  "monthlySummary": {
    "month": "2026-05",
    "availableIncome": 250000,
    "fixedCostTotal": 120000,
    "expenseTotal": 91580,
    "remainingAmount": 38420
  }
}
```

### 一覧取得

一覧APIは必ず `items` で配列を返す。

該当データがない場合は `items: []` を返す。

```json
{
  "items": []
}
```

将来paginationを追加する場合も、`items` のキー名は変えない。

### 集計取得

集計APIはendpointの対象が分かるキーで包む。

```json
{
  "monthlySummary": {
    "month": "2026-05",
    "remainingAmount": 105000
  }
}
```

### 削除

削除APIは成功メッセージを返す。

```json
{
  "message": "削除しました"
}
```

---

## エラーレスポンス

エラー時は以下の形式で返す。

```json
{
  "message": "エラーメッセージ"
}
```

必要に応じて詳細エラーを追加する。

```json
{
  "message": "入力内容が不正です",
  "details": [
    {
      "field": "amount",
      "message": "金額は1円以上で入力してください"
    }
  ]
}
```

---

## HTTPステータス

| ステータス | 用途 |
|---|---|
| 200 | 取得・更新・削除成功 |
| 201 | 作成成功 |
| 400 | リクエスト不正 |
| 401 | 未認証 |
| 403 | 権限なし |
| 404 | 対象データなし |
| 500 | サーバーエラー |

---

## 代表的なエラーメッセージ

| ケース | message |
|---|---|
| 未認証 | `認証が必要です` |
| 権限なし | `操作する権限がありません` |
| 対象データなし | `対象データが見つかりません` |
| バリデーションエラー | `入力内容が不正です` |
| サーバーエラー | `サーバーでエラーが発生しました` |

---

## レスポンスに含めないもの

`userId` は、UI表示や仕様上必要な場合を除きレスポンスに含めない。

認証済みユーザー本人のデータであることは、Authorizationヘッダーとバックエンド側の認可で保証する。

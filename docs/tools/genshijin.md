# genshijin Usage

## 目的

genshijinは、AIエージェントの返答を短く圧縮するためのコミュニケーションモードです。

技術的な正確性は維持しつつ、敬語、前置き、冗長な説明を削り、短い指示・確認・報告に寄せます。

## 強度レベル

### 丁寧

敬語は残し、前置きやぼかしだけ削ります。

```txt
/genshijin 丁寧
```

### 通常

デフォルトです。

敬語を落とし、体言止め、短文、助詞省略を使います。

```txt
/genshijin 通常
```

### 極限

キーワード中心にさらに圧縮します。

識別子、関数名、API名、エラーメッセージは省略しません。

```txt
/genshijin 極限
```

## 解除方法

以下のどちらかを指示します。

```txt
原始人やめて
```

```txt
通常モード
```

## 使いどころ

- 実装中の進捗確認
- エラー原因の短い説明
- 次にやる作業の確認
- PRレビュー指摘の要点整理
- 長い説明が不要な反復作業

## 使わない方がよい場面

- 初心者向けに丁寧な背景説明が必要なとき
- 仕様の認識合わせで誤読を避けたいとき
- 破壊的操作やセキュリティ警告など、順序と条件を明確に書く必要があるとき
- 複雑なSQL、正規表現、手順書など、圧縮で意味が崩れやすいとき

## 指示例

### 通常の作業を短く進めたい場合

```txt
/genshijin 通常
mainを最新化してfeatureブランチを切り、固定費画面のUIを修正して。
```

### 最小限の報告だけ欲しい場合

```txt
/genshijin 極限
検証結果だけ教えて。
```

### 説明は短く、敬語は残したい場合

```txt
/genshijin 丁寧
このエラーの原因と修正方針を短く教えて。
```

## このプロジェクトでの注意

genshijinを使っても、以下のプロジェクト規約は維持します。

- `main` からfeatureブランチを切る
- 実装前に該当docsを確認する
- `sx` は `*.styles.ts` に置く
- componentsは表示とprops callback呼び出しに寄せる
- hooks/usecasesへフォーム制御、保存判断、API mutationを寄せる
- コメント規約を守る
- 検証結果を必ず明記する

返答は短くしても、実装品質ゲートは省略しません。

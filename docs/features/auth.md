# Auth Feature

## 目的

ログインした本人だけが家計簿アプリを利用できるようにする。

MVPでは新規登録画面は作らず、Supabase Auth側でユーザーを事前作成する。

---

## 画面

### ログイン画面

- パス: `/login`
- メールアドレスとパスワードでログインする
- ログイン成功後は `/` に遷移する
- ログイン失敗時はエラーメッセージを表示する
- 新規登録導線は表示しない
- ログイン済み状態で `/login` にアクセスした場合は `/` にリダイレクトする

---

## 認証方針

- Supabase Authを利用する
- 未ログイン状態ではアプリ画面を表示しない
- 未ログインの場合は `/login` にリダイレクトする
- APIはすべて認証必須とする
- フロントエンドはAPIリクエスト時にaccess tokenを送信する
- バックエンドはaccess tokenを検証し、アプリ側のuserIDを決定する
- `user_id` はrequest body、query、URL parameterから受け取らない

---

## Supabase Auth連携

- ログイン
- ログアウト
- アクセストークン取得
- Go APIへトークン送信
- Go API側でトークン検証
- usersレコードの取得または作成
- user_id紐づけ
- 未ログイン時のルートガード

---

## MVP完了条件

- メールアドレスとパスワードでログインできる
- ログイン成功後に `/` へ遷移する
- 未ログイン状態でアプリ画面へアクセスすると `/login` へリダイレクトする
- ログイン済み状態で `/login` へアクセスすると `/` へリダイレクトする
- APIリクエストにaccess tokenを付与できる
- Go API側でaccess tokenを検証し、アプリ側のuserIDを決定できる
- ログアウトできる
- ログアウト後はアプリ画面を表示せず `/login` へ遷移する

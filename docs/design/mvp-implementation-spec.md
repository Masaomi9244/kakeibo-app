# MVP Design Implementation Spec

## 目的

Figma Make案をもとにしつつ、`docs/product/scope.md` に忠実なMVP UIを実装するためのdesign specを定義する。

このspecは実装前の判断ブレを減らすためのものであり、最終的な正は以下の順で判断する。

1. `docs/product/scope.md`
2. `docs/features/*.md`
3. `docs/front-end/spec.md`
4. `docs/standards/front-end/index.md`
5. このdesign spec

Figma Makeは2026年6月1日まで追加修正に期待しない。
2026-05-05時点の修正版スクリーンショットを見た目の基準とし、仕様不足やscope差分はこのspecで補正する。

## デザイン原則

- 「今月あといくら使えるか」を最優先で見せる
- 出費登録はトップ画面から金額のみで完了させる
- カテゴリ分析ではなく、残額把握を中心にする
- 毎日使う操作画面として、派手さよりも迷わなさを優先する
- スマホで片手操作しやすいことを優先する
- PCでは情報量を増やすが、MVPの画面数は増やさない
- Next.js、MUI、既存のfrontend standardsに合わせて実装する
- Figma Makeのshadcn / Tailwindコードは直接移植しない
- Figma Makeのスクリーンショットは見た目の参考に限定し、表示項目は必ずscopeに合わせて削る
- Figma Make上のURL表記より、`docs/product/scope.md` のパスを優先する

## ナビゲーション

### MVPで表示するナビ項目

- ホーム
- 収入
- 固定費
- カレンダー
- 年間

### MVPで表示しないナビ項目

- 支出
- 設定
- カテゴリ
- 予算
- 通知
- 共有

### ログアウト導線

ログアウトはMVP完了条件に含める。

ただし、設定画面は作らず、以下のいずれかに配置する。

- PC: サイドナビ下部
- スマホ: 下部ナビとは別のヘッダー内メニュー

2026-05-05時点の修正版スクリーンショットでは、PCはサイドナビ下部、スマホはヘッダー右側のログアウトアイコンとして表示されている。
実装でもこの方向を採用する。

## 画面別spec

### ログイン画面

パス: `/login`

表示するもの:

- アプリ名
- メールアドレス入力
- パスワード入力
- ログインボタン
- ログイン失敗時のエラー表示

表示しないもの:

- 新規登録リンク
- アカウント作成ボタン
- パスワード再設定導線
- SNSログイン

状態:

- 通常
- 入力エラー
- ログイン中
- ログイン失敗

### トップ画面

パス: `/`

最重要画面。
Figma Makeの `HomePage` をベースにするが、`ExpensesPage` の役割をここへ統合する。

表示するもの:

- 今月残り使える金額
- 出費金額入力欄
- 出費登録後のUndo Snackbar
- 今日の出費一覧
- 今月の簡易内訳
  - 使える収入
  - 固定費
  - 出費

出費入力:

- 金額のみ入力する
- カテゴリは入力しない
- メモはMVPでは入力しない
- フォーカスアウトで登録する
- Enterキーでも登録する
- 登録ボタンは表示しない
- 登録成功後に入力欄をクリアする
- 登録成功後に3秒間Undoを表示する
- 保存中は二重登録しない
- 保存失敗時は入力値を残し、入力欄の近くにエラーを表示する

今日の出費一覧:

- Asia/Tokyo基準の今日の出費のみ表示する
- 表示項目は金額と登録時刻に絞る
- カテゴリ、カテゴリ文字アイコン、メモは表示しない
- `すべて見る` 導線は表示しない

状態:

- 通常
- 出費0件
- 保存中
- 保存失敗
- Undo表示中

2026-05-05時点の修正版スクリーンショットではUndo表示は確認できない。
ただし、MVP完了条件に含まれるため実装では必ず追加する。

### 収入画面

パス: `/incomes`

Figma Make修正版のURL表示が `/income` でも、実装パスは `/incomes` とする。

表示するもの:

- 収入一覧
- 収入追加フォーム
- 金額
- 収入日
- メモ
- 今月使えるお金に含めるか
- 編集
- 削除

表示しないもの:

- 収入カテゴリ
- 収入カテゴリ別集計

状態:

- 通常
- 空データ
- 読み込み中
- 保存中
- 入力エラー
- 削除確認

### 固定費画面

パス: `/fixed-costs`

表示するもの:

- 固定費一覧
- 固定費追加フォーム
- 固定費名
- 金額
- 開始月
- 有効かどうか
- 編集
- 削除

表示しないもの:

- 終了月
- 固定費カテゴリ
- 固定費の履歴管理

状態:

- 通常
- 空データ
- 読み込み中
- 保存中
- 入力エラー
- 削除確認

### カレンダー画面

パス: `/calendar`

表示するもの:

- 対象月
- 月移動
- カレンダー
- 各日の出費合計
- 各日の終了時点残額
- 選択日の出費明細
- 1日の目安
- 平均出費/日

表示しないもの:

- カテゴリ別表示
- 予算カテゴリ別表示

補足:

- PC幅では、各日セルに出費合計と終了時点残額を表示することを優先する
- スマホ幅では、各日セルに出費合計を表示し、終了時点残額は選択日の詳細領域に表示してよい
- スマホ幅で日付、出費合計、終了時点残額をすべてセル内に詰め込み、文字つぶれや重なりが起きる場合は避ける
- 選択日の詳細領域には、日付、合計、日終了時点残額、出費明細を表示する
- 1日の目安と平均出費/日は補助情報とし、日別残額より優先しない

2026-05-05時点の修正版スクリーンショットでは、選択日の出費明細カードに日付、合計、時刻、金額が表示されている。
実装ではこの構成を採用し、日終了時点残額を追加する。

状態:

- 通常
- 出費0件
- 読み込み中
- エラー

### 年間サマリー画面

パス: `/annual-summary`

Figma Make修正版のURL表示が `/annual` でも、実装パスは `/annual-summary` とする。

表示するもの:

- 年間全収入
- 年間使える収入
- 年間貯める収入
- 年間固定費
- 年間出費
- 年間実収支
- 年間生活費残り
- 月別推移グラフ
- 月別サマリー一覧

表示しないもの:

- 出費カテゴリ割合グラフ
- カテゴリ別ランキング
- 予算カテゴリ分析
- 総貯蓄

月別サマリー一覧の表示項目:

- 全収入
- 使える収入
- 貯める収入
- 固定費
- 出費
- 実収支
- 生活費残り

補足:

- `支出が多かった月`、`支出が少なかった月` は月別集計の補助カードとして表示してよい
- `今年の傾向` を表示する場合もカテゴリ分析には寄せず、月次の収支傾向に限定する
- PC幅では月別サマリー一覧をテーブル表示してよい
- スマホ幅では月別サマリー一覧をカード表示、または横スクロール可能な表にする
- スマホ幅でテーブルの列名が縦書きのように崩れる状態は不可とする
- グラフの凡例が下部ナビや次のカードに隠れないようにする

2026-05-05時点の修正版スクリーンショットでは、スマホ幅の月別サマリー一覧が窮屈に見える。
実装では可読性を優先し、スマホ向けに表示形式を補正する。

状態:

- 通常
- 年間データなし
- 読み込み中
- エラー

## レスポンシブ方針

### スマホ

- 下部ナビを使う
- トップ画面の出費入力をファーストビューに置く
- 金額入力は数字キーボードを出しやすくする
- カードは縦積みにする
- 主要操作は親指で届く位置に置く

### PC

- サイドナビを使う
- トップ画面は2カラムまでにする
- 残額、入力欄、今日の出費、今月内訳が同時に見えるようにする
- 画面幅が広くても情報を横に広げすぎない

## 状態表現

共通で以下を用意する。

- Loading
- Empty
- Error
- Input error
- Saving
- Disabled

Figma Makeの `EmptyState`、`ErrorMessage`、`LoadingSpinner` は考え方として採用する。
ただし実装はMUIコンポーネントで作る。

## コンポーネント分解

実装時のコンポーネント分解は `docs/design/component-breakdown.md` を正とする。

このdesign specは画面の仕様を定義し、`component-breakdown.md` は実装単位、props責務、hooks候補、作らないコンポーネントを定義する。

## 実装時に使わないFigma Make要素

- shadcn / Tailwind前提の実装コード
- 支出専用ページ
- 支出登録ナビ
- 設定ページ
- カテゴリ選択UI
- 出費メモ入力
- 登録ボタン付きの出費フォーム
- カテゴリ別集計UI
- デモ用ログイン文言
- マーケティングサイト風のヒーロー表現

## 実装時の優先順位

1. ログイン画面
2. トップ画面
3. 収入画面
4. 固定費画面
5. カレンダー画面
6. 年間サマリー画面
7. PWA表示設定

年間サマリーはMVP対象だが、実装順序は後半とする。

## 実装前チェック

実装開始前に以下を確認する。

- `/expenses` を追加していない
- `/settings` を追加していない
- 出費入力にカテゴリを追加していない
- 出費入力にメモを追加していない
- 出費入力に登録ボタンを追加していない
- 今日の出費一覧にカテゴリを表示していない
- 年間サマリーにカテゴリ割合を追加していない
- カレンダーで日別終了時点残額を確認できる
- 年間サマリーに月別サマリー一覧がある
- ログイン画面に新規登録導線を追加していない
- ログアウト導線はある
- スマホのトップ画面で残額と出費入力がすぐ見える
- `docs/product/scope.md` の「やらないこと」と矛盾していない
